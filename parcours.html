Compétence : Administrer les réseaux et l'Internet

AC 11.04 : Maîtriser les rôles et les principes fondamentaux des systèmes d’exploitation

Ce que j'ai fait : J'ai conçu et déployé une infrastructure réseau centralisée sous Windows Server 2016. J'ai mis en place une architecture Active Directory avec des groupes distincts (Élèves/Profs), géré des points de montage NTFS, et automatisé le déploiement de logiciels via des GPO. J'ai également créé un script PowerShell pour automatiser la création des comptes.

Pourquoi je l'ai fait : L'objectif était de centraliser l'authentification, de sécuriser les données via le "Moindre Privilège", et d'automatiser les tâches répétitives pour déployer les ressources de manière industrielle et sécurisée.

Comment je l'ai fait : J'ai utilisé l'outil Utilisateurs et ordinateurs Active Directory. J'ai configuré les partages réseau en "Contrôle total" pour laisser la couche locale (NTFS) gérer les permissions restrictives. J'ai également développé un script script.ps1 utilisant les cmdlets New-ADUser, New-Item et l'outil icacls.

Mes difficultés : Comprendre que la permission finale est la plus restrictive entre celle du partage et celle du NTFS a été complexe. La syntaxe de l'outil icacls dans PowerShell a également demandé de la rigueur. J'ai dû forcer les mises à jour de stratégies avec gpupdate /force.

Ce que j'en ai appris : J'ai saisi la différence fondamentale entre permissions réseau et locales. J'ai surtout pris conscience de l'importance de l'industrialisation (scripts, GPO) pour garantir la conformité d'un parc informatique.

Ce que je ferais autrement : Pour le script PowerShell, je ferais évoluer mon code pour qu'il lise un fichier .csv contenant les données de dizaines d'utilisateurs, permettant d'importer une promotion entière d'un seul coup.

Compétence : Connecter les entreprises et les usagers

AC 12.04 : Connecter les systèmes de ToIP (Téléphonie sur IP)

Ce que j'ai fait : J'ai déployé et configuré un serveur de téléphonie IPBX sous Asterisk (Linux Alpine). J'ai configuré des téléphones physiques de différentes marques : des postes Fanvil (via leur interface web graphique) et des postes Cisco (via auto-provisioning avec un serveur TFTP). J'ai également paramétré un plan de numérotation avec la gestion de la messagerie vocale (Voicemail) et un serveur vocal interactif (IVR).

Pourquoi je l'ai fait : L'objectif était de mettre en place une infrastructure de communication complète permettant aux collaborateurs d'une entreprise de s'appeler en interne, de laisser des messages en cas de non-réponse, et de filtrer les appels entrants via un menu vocal automatisé, tout en garantissant l'interopérabilité entre différents constructeurs (Cisco et Fanvil).

Comment je l'ai fait : Dans Asterisk, j'ai manipulé le fichier pjsip.conf pour déclarer les comptes SIP (endpoints, auth, aor) et le fichier extensions.conf pour définir le comportement du réseau (le Dialplan). Pour les téléphones Fanvil, j'ai renseigné manuellement l'IP du serveur SIP et l'authentification dans l'interface web. Pour les postes Cisco, j'ai installé un service in.tftpd et configuré des fichiers XML (ex: SEP.cnf.xml) pour qu'ils récupèrent automatiquement leur configuration au démarrage sur le réseau.

Mes difficultés : La syntaxe des fichiers de configuration Asterisk est très stricte. J'ai rencontré des difficultés avec la gestion des adresses IP et l'enregistrement des téléphones (problèmes de bind ou d'identifiants incorrects dans pjsip.conf). De plus, l'auto-provisioning des téléphones Cisco exigeait que le nom du fichier XML contienne la bonne adresse MAC sans aucune erreur de formatage.

Ce que j'en ai appris : J'ai compris le fonctionnement concret du protocole SIP (utilisant le port UDP 5060) pour l'établissement des sessions. J'ai également saisi l'énorme avantage de l'auto-provisioning (TFTP/XML) en milieu professionnel par rapport à une configuration manuelle poste par poste via interface web.

Ce que je ferais autrement : Plutôt que de copier et de modifier manuellement le fichier XML SEP_MAC-CISCO_.cnf.xml pour chaque poste Cisco, je développerais un script Bash ou Python qui prendrait en entrée un fichier CSV (contenant les adresses MAC et les numéros d'extension) pour générer automatiquement tous les fichiers XML dans le dossier /var/tftpboot.

📸 Traces ciblées :

Fichier extensions.conf (Dialplan)

Fichier pjsip.conf (Enregistrement SIP)

AC 13.05 & 13.06 : Traitement de données et Environnement Collaboratif (Python)

Ce que j'ai fait : Dans le cadre de la SAE 1.05, j'ai développé un outil logiciel en Python permettant d'analyser des données en format Open Data. Concrètement, j'ai créé un script modulaire capable de lire un fichier de données de clubs sportifs (clubs-data-2023.csv), de traiter ces informations, et de générer des visualisations sous forme de graphiques en ASCII directement dans le terminal.

Pourquoi je l'ai fait : L'objectif était d'automatiser le traitement d'un grand volume de données brutes pour les rendre lisibles et exploitables. Il s'agissait également d'apprendre à travailler de manière professionnelle en découpant le code en plusieurs fichiers distincts.

Comment je l'ai fait : J'ai structuré mon projet Python de manière modulaire avec un main.py qui sert de chef d'orchestre, appelant des fonctions d'analyse réparties dans make_chart.py et create_ascii_chart.py. J'ai utilisé les structures de données natives de Python (listes, dictionnaires) pour manipuler les données extraites du fichier CSV. Ce projet a été géré de manière collaborative en utilisant GitLab.

Mes difficultés : La gestion du formatage du fichier CSV a été un défi (gestion des séparateurs, encodage des caractères). De plus, comprendre comment importer correctement mes modules d'un fichier à un autre a demandé des recherches pour éviter les erreurs d'importation.

Ce que j'en ai appris : J'ai acquis des bases solides en algorithmique et en manipulation de données en Python. J'ai surtout appris l'importance cruciale d'un code propre, commenté et modulaire.

Ce que je ferais autrement : Au lieu de parser le fichier CSV manuellement avec les fonctions de base de Python, j'utiliserais à l'avenir une bibliothèque spécialisée comme Pandas pour écrire un code beaucoup plus robuste.

#!/usr/bin/python3

import pandas as pd
import requests
import json
import os
from subprocess import run
import argparse
from create_ascii_chart import draw

parser = argparse.ArgumentParser(description = "Download .cvs and get top N (default 10) groups")
parser.add_argument("-l", "--link", type = str, nargs = "?", default = "https://www.data.gouv.fr/api/1/datasets/r/880cca20-3300-4f93-8b67-4de515b42dc9", help = "Use custom link to download dataset")
parser.add_argument("-n", "--number", type = int, nargs = "?", default = 10, help = "Use custom number of groups to chart")
parser.add_argument("-a", "--ascii", type = int, nargs = "?", default = 1, help = "Use -a0 to turn off ascii chart")
parser.add_argument("-q", "--quickchart", type = int, nargs = "?", default = 1, help = "Use -q0 to turn off QuickChart")
parser.add_argument("-o", "--open_quickchart", type = int, nargs = "?", default = 1, help = "Use -o0 to prevent script from opening QuickChart image automatically.")
args = parser.parse_args()

do_ascii = bool(args.ascii)
do_quickchart = bool(args.quickchart)

# ÉTAPE 1 : CHARGEMENT DES DONNÉES
nom_fichier = 'clubs-data-2023.csv'
adresse_fichier = args.link

print(f" Chargement du fichier {nom_fichier}...")

data = requests.get(adresse_fichier)
with open(nom_fichier, "wb") as file:
    file.write(data.content)

# On vérifie si le fichier existe
if not os.path.exists(nom_fichier):
    print(f" ERREUR : Le fichier '{nom_fichier}' est introuvable !")
    print(" Assure-toi qu'il est bien dans le même dossier que ce script.")
    exit()

try:
    # Le fichier utilise des points-virgules comme séparateur
    df = pd.read_csv(nom_fichier, sep=';')
    print(" Fichier chargé avec succès !")
    print(f"   Total de lignes : {len(df)}")
    print(f"   Colonnes : {', '.join(df.columns)}")
except Exception as e:
    print(f" Erreur lors de la lecture du CSV : {e}")
    exit()

# ÉTAPE 2 : CALCULS (Top N des Fédérations)
print("\n Calcul des statistiques en cours...")

# On groupe par "Fédération" et on additionne la colonne "Total" (Clubs + EPA)
# On trie du plus grand au plus petit et on garde les N premiers

groups_number = args.number

top = df.groupby('Fédération')['Total'].sum().sort_values(ascending=False).head(groups_number)

top_list = [list(dict(top).keys()), list(top)]


if do_quickchart:
    # On prépare les listes pour le graphique
    noms_fedes = top.index.tolist()   # Axe X (Noms)
    nb_structures = top.values.tolist() # Axe Y (Chiffres)

    print(" Top 3 des fédérations trouvées :")
    for i in range(3):
        print(f"   {i+1}. {noms_fedes[i]} ({nb_structures[i]} structures)")

    # ÉTAPE 3 : GÉNÉRATION DU GRAPHIQUE (QuickChart.io)
    print("\n Création du graphique...")

    # Configuration JSON pour QuickChart
    chart_config = {
        "type": "bar",
        "data": {
            "labels": noms_fedes, # Les noms des fédés
            "datasets": [{
                "label": "Nombre de Clubs & Établissements (2023)",
                "data": nb_structures, # Les chiffres
                "backgroundColor": "rgba(54, 162, 235, 0.6)", # Bleu joli
                "borderColor": "rgba(54, 162, 235, 1)",
                "borderWidth": 1
            }]
        },
        "options": {
            "title": {
                "display": True,
                "text": "Top 10 des Fédérations Sportives (Nombre de structures)"
            },
            "scales": {
                "yAxes": [{
                    "ticks": {
                        "beginAtZero": True
                    }
                }]
            }
        }
    }

    # Envoi à l'API
    url_api = "https://quickchart.io/chart/create"
    payload = {
        "chart": chart_config,
        "width": 800,
        "height": 500,
        "backgroundColor": "white"
    }

    try:
        response = requests.post(url_api, json=payload)
        
        if response.status_code == 200:
            resultat = response.json()
            print("\n SUCCÈS TOTAL ! ")
            print("Clique sur ce lien pour voir ton graphique :")
            quickchart_link = resultat['url']
            print(quickchart_link)
        else:
            print(" Erreur de l'API QuickChart :", response.text)

    except Exception as e:
        print(f" Erreur de connexion Internet : {e}")

if do_ascii:       #Printing ASCII chart
    print('\n') 
    draw(top_list)

#Downloading Quickchart
if do_quickchart and args.open_quickchart:
    qc_chart = requests.get(quickchart_link)
    with open('QuickChart_chart.png', 'wb') as file:
        file.write(qc_chart.content)
        file.close()
    run(['xdg-open', 'QuickChart_chart.png'])
